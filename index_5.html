// about.md

I don't just find bugs.
I design for fewer of them.

Over six years, I've tested ERP platforms, banking and financial workflows, e-commerce checkouts, education systems, CRMs, and loyalty platforms — environments where a missed edge case has real consequences. That's taught me to think in business workflows first, then translate them into structured test coverage.

At Safe IMS, I own REST API validation across integration-heavy ERP systems supporting 500+ users — authentication flows, payload structures, error handling — and run weekly Newman regression suites as the last checkpoint before issues reach production. Alongside that, I've started writing Playwright automation in Java, extending UI regression coverage as part of a lean two-person QA team.

I'm deepening my work in Java and the Page Object Model, and building CI/CD-integrated automation frameworks — extending what I do as a "QA Engineer who automates" into a full Automation QA Engineer who designs the framework other testers build on.

DOMAIN EXPERIENCE

TESTING PHILOSOPHY

Prevent before you detect. A good test case answers a "what if" the requirements doc never asked — that's where exploratory testing earns its place beside structured suites.

CURRENT FOCUS
